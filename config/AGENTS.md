# Kustomize Configuration

`config/` holds kustomize bases and overlays for deploying the operator. Most CRD/RBAC/webhook artifacts are generated by `controller-gen` via `make manifests` — do not edit them by hand.

## Layout

- `crd/bases/`: generated CRD YAMLs (one per CRD, named `{group}.{resource}.yaml`). Generated by `make manifests`.
- `crd/patches/`: kubebuilder scaffold webhook/cainjection patch templates (hand-written, mostly commented out).
- `rbac/role.yaml`: generated `manager-role` ClusterRole. All other RBAC files (service account, bindings, editor/viewer roles) are hand-written.
- `webhook/manifests.yaml`: generated MutatingWebhookConfiguration + ValidatingWebhookConfiguration.
- `manager/`: hand-written Namespace + Deployment scaffold and controller-manager config.
- `default/`: the main overlay combining crd + rbac + manager + webhook + certmanager. Applies `namePrefix: kubeblocks-` and `namespace: kubeblocks-system`.
- `certmanager/`, `prometheus/`, `samples/`: certificate, ServiceMonitor, and example CR manifests.

## Editing Rules

- Generated files (do not hand-edit): `crd/bases/*.yaml`, `rbac/role.yaml`, `webhook/manifests.yaml`. Regenerate with `make manifests`.
- After changing API types in `apis/`, run `make generate` (deepcopy) and `make manifests` (CRD/RBAC/webhook + sync to `deploy/helm/`).
- `make manifests` also copies generated CRDs to `deploy/helm/crds/` and `config/rbac/role.yaml` to `deploy/helm/config/rbac/role.yaml` — keep this sync path intact.
- `make install` applies only CRDs (`kustomize build config/crd`); `make deploy` applies the full overlay (`kustomize build config/default`).
- Hand-written files: all `kustomization.yaml`, `kustomizeconfig.yaml`, patches, `manager/manager.yaml`, `webhook/service.yaml`, `certmanager/*`, `prometheus/*`, `samples/*`.
- CRD file naming: `{full-group-domain}_{resource-plural}.yaml` (e.g. `apps.kubeblocks.io_clusters.yaml`).
- RBAC editor/viewer role naming: `{group-singular}_{resource}_{editor|viewer}_role.yaml`.
