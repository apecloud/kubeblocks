# Helm Chart

`deploy/helm/` is the KubeBlocks Helm chart (`CHART_PATH = deploy/helm` in the Makefile). It packages the operator deployment, CRDs, RBAC, webhooks, addons, and storage providers.

## Layout

- `Chart.yaml`, `values.yaml`: chart metadata and values (630 lines with `@param` docs).
- `crds/`: generated CRDs copied from `config/crd/bases/` by `make manifests`. The `.helmignore` excludes `crds/` so CRDs are **not** installed by `helm install` — install separately via `make install`.
- `crds/snapshot/`: hand-curated external VolumeSnapshot CRDs (not generated by this project).
- `config/rbac/role.yaml`: generated RBAC copied from `config/rbac/role.yaml`. Consumed at render time by `templates/rbac/clusterrole.yaml` via `.Files.Lines`.
- `templates/`: hand-written Helm templates. `_helpers.tpl`, `_addon.tpl`, `_dataprotection.tpl` are named-template helpers.
- `templates/addons/`: per-engine Addon CR templates (`{engine}-addon.yaml`), controlled by `autoInstalledAddons` values.
- `templates/rbac/`: RBAC templates. `clusterrole.yaml` bridges generated RBAC via `.Files.Lines "config/rbac/role.yaml"`.
- `templates/admission/`: webhook configuration with self-signed cert generation (`genCA`/`genSignedCert`).
- `templates/storageprovider/`: StorageProvider CRs per backend (`s3.yaml`, `oss.yaml`, `nfs.yaml`, etc.).
- `scripts/`: hand-written helm hook scripts (pre/post-install addon hooks).

## Editing Rules

- Do not hand-edit `crds/*.yaml` (except `crds/snapshot/`) or `config/rbac/role.yaml` — they are generated by `make manifests`.
- When adding a new CRD, `make manifests` auto-syncs it to `crds/`; verify the `.helmignore` still excludes the directory.
- When modifying RBAC rules via `+kubebuilder:rbac` markers, `make manifests` regenerates `config/rbac/role.yaml` and syncs it here — do not edit `templates/rbac/clusterrole.yaml` rules directly.
- `values.yaml` uses `@param` documentation comments — maintain this convention when adding values.
- Addon templates use `buildAddonCR` from `_addon.tpl`; follow the existing `{engine}-addon.yaml` pattern.
- Hook scripts in `scripts/` are referenced by Job templates via `{{ .Files.Get }}` — keep filenames consistent.
